## 🔧 Windows Server 관리
> Windows 서버 관리와 초기화에 관한 핵심 도구들 알아보기

### Initialization Tool 🛠️
- **기본 개요**:
  ```
  - Windows 이미지 생성 전 SID 초기화 도구
  - Windows 2012/2016/2019 운영체제 지원
  - 서버 배포 자동화 지원
  ```

- **주요 목적**:
  ```
  - Active Directory 구성 시 SQL 인증 오류 방지
  - 서버 SID 초기화 자동화
  - 배포 프로세스 표준화
  ```

- **Tool 비교**:
  ```
  1. Initialization Tool
    ㄴ SID 초기화 특화
    ㄴ SQL 인증 오류 방지
    ㄴ 기본 초기화 작업
  
  2. Sysprep
    ㄴ 전체 시스템 초기화
    ㄴ OS 복제 준비
    ㄴ 전체 설정 리셋
  ```

- **실행 절차**:
  ```
  1. PowerShell 관리자 실행
  2. 초기화 도구 다운로드
  3. MSSQL 설치 확인
  4. 초기화 작업 수행
  ```

## 💾 Linux System Administration
> Linux 스토리지 관리와 LVM 구성 이해하기

### LVM (Logical Volume Manager) 📀
- **핵심 특징**:
  ```
  1. 유연한 디스크 관리
    ㄴ 동적 용량 조절
    ㄴ 논리적 볼륨 관리
  
  2. 무중단 확장
    ㄴ 운영중 용량 확장
    ㄴ 서비스 연속성 보장
  
  3. 성능 최적화
    ㄴ RAID 유사 구성
    ㄴ I/O 분산 처리
  ```

- **구성 절차**:
  ```
  1. fdisk
    ㄴ /dev/sdb1 파티션 생성
    ㄴ 디스크 파티셔닝
  
  2. pvcreate
    ㄴ 물리 볼륨 설정
    ㄴ 파티션 초기화
  
  3. vgcreate
    ㄴ 볼륨 그룹 생성
    ㄴ PV 통합 관리
  
  4. lvcreate
    ㄴ 논리 볼륨 생성
    ㄴ 파일시스템 준비
  ```

- **파티션 유형**:
  ```
  - 83: Linux 기본 파티션
  - e6: RAID 구성용
  - 80: 부팅 파티션
  - 8e: LVM 전용
  ```

## 🗄️ NAS (Network Attached Storage)
> 네트워크 스토리지 시스템의 이해

### 기본 구성
- **핵심 개념**:
  ```
  - 네트워크 기반 스토리지
  - 중앙 집중식 데이터 관리
  - 다중 사용자 지원 구조
  ```

- **스토리지 비교**:
  ```
  1. NAS
    ㄴ 파일 레벨 접근
    ㄴ TCP/IP 네트워크
    ㄴ 높은 확장성

  2. DAS
    ㄴ 직접 연결 방식
    ㄴ 전용 케이블
    ㄴ 제한적 확장

  3. SAN
    ㄴ 블록 레벨 접근
    ㄴ 전용 네트워크
    ㄴ 고성능 보장
  ```
  
### 핵심 기능
- **데이터 보호**:
  ```
  - RAID 구성
    ㄴ 데이터 중복성
    ㄴ 성능 최적화
  
  - 스냅샷 지원
    ㄴ 시점 복구
    ㄴ 백업 자동화
  
  - 실시간 백업
    ㄴ 데이터 동기화
    ㄴ 장애 대비
  ```

- **성능 관리**:
  ```
  - IOPS 모니터링
    ㄴ 성능 측정
    ㄴ 병목 감지
  
  - 대역폭 관리
    ㄴ 트래픽 제어
    ㄴ QoS 설정
  
  - 캐시 최적화
    ㄴ 접근 속도 향상
    ㄴ 부하 분산
  ```

## ☁️ Cloud Infrastructure
> 클라우드 환경 구성요소와 아키텍처 이해하기

### VPC 환경의 Zone 종속성 🌐

- **서버 (VM)**:
  ```
  - 특징: Zone 종속 O
  - 역할: 가상 컴퓨팅 자원 제공
  - 용도: 애플리케이션 실행, 웹 호스팅
  
  클라우드별 특징:
  - AWS: EC2는 AZ 내 실행
  - GCP: VM은 Zone에 배치
  - Azure: VM은 Zone에 할당
  - NCP: Compute는 Zone 기반
  ```

- **로드밸런서**:
  ```
  - 특징: Zone 종속 X
  - 역할: 트래픽 분산 처리
  - 범위: 다중 AZ 지원
  
  클라우드별 특징:
  - AWS: ELB로 다중 AZ 지원
  - GCP: 글로벌 부하분산
  - Azure: 다중 Zone 분산
  - NCP: L7 다중 Zone 지원
  ```

## 🎯  🎯 Kubernetes Architecture
> 쿠버네티스 구성요소와 작동 원리 이해하기

### 노드 구성 및 역할
- **Master Node (Control Plane)**:
  ```
  1. 핵심 구성요소
    ㄴ API Server: 클러스터 API 제공
    ㄴ Scheduler: 파드 스케줄링
    ㄴ Controller Manager: 상태 관리
    ㄴ etcd: 구성 데이터 저장

  2. 주요 기능
    ㄴ 클러스터 제어
    ㄴ 워크로드 관리
    ㄴ 상태 모니터링
  ```

- **Worker Node**:
  ```
  1. 핵심 구성요소
    ㄴ Kubelet: 컨테이너 관리
    ㄴ Kube Proxy: 네트워크 관리
    ㄴ Container Runtime: 컨테이너 실행

  2. 주요 기능
    ㄴ 컨테이너 실행
    ㄴ 네트워크 연결
    ㄴ 볼륨 관리
  ```

### 클라우드 관리 형태
- **클라우드별 특징**:
  ```
  1. AWS EKS
    ㄴ 마스터 노드 자동화
    ㄴ Control Plane 관리형
    ㄴ Worker Node 사용자 관리

  2. Google GKE
    ㄴ 자동 클러스터 운영
    ㄴ 마스터-etcd 관리형
    ㄴ Worker Node 유연성
  
  3. NCP NKS
    ㄴ 관리형 마스터 노드
    ㄴ 자동 클러스터 확장
    ㄴ 통합 모니터링
  ```

## 🖥️ 네이버 클라우드 서버
> 네이버 클라우드 서버 구성과 관리 방안

### 서버 유형별 특징
- **서버 종류**:
  ```
  1. 표준 서버
    ㄴ 일반 워크로드
    ㄴ KVM/Xen 하이퍼바이저
    ㄴ 범용 성능

  2. SSD 서버
    ㄴ 고성능 I/O
    ㄴ KVM 기반
    ㄴ 데이터베이스 최적화

  3. GPU 서버
    ㄴ AI/ML 워크로드
    ㄴ CUDA 지원
    ㄴ 고성능 연산

  4. 베어메탈 서버
    ㄴ 물리 서버 제공
    ㄴ 하이퍼바이저 없음
    ㄴ 최고 성능
  ```

### 하이퍼바이저 관리
- **유형별 특징**:
  ```
  1. KVM
    ㄴ 리눅스 기반
    ㄴ I/O 성능 최적화
    ㄴ 동적 리소스 관리

  2. Xen
    ㄴ 안정성 중심
    ㄴ 레거시 지원
    ㄴ 메모리 효율성
  ```

### VPC 구성 관리
- **네트워크 설계**:
  ```
  1. 서브넷 구성
    ㄴ Public: 외부 접근용
    ㄴ Private: 내부 전용
    ㄴ Zone별 설계

  2. 보안 설정
    ㄴ 보안그룹: 인스턴스 방화벽
    ㄴ ACL: 서브넷 접근제어
    ㄴ SSL-VPN: 원격접속
  ```

### 서버 관리
- **생성 프로세스**:
  ```
  1. 기본 구성
    ㄴ VPC/서브넷 선택
    ㄴ 보안그룹 설정
    ㄴ 스펙 선택

  2. 이미지 관리
    ㄴ 표준 이미지
    ㄴ 커스텀 이미지
    ㄴ 공유 이미지
  ```

### 성능 최적화
- **서버별 최적화**:
  ```
  1. SSD 서버
    ㄴ IOPS 설정
    ㄴ 볼륨 최적화
    ㄴ 캐시 구성

  2. GPU 서버
    ㄴ CUDA 설정
    ㄴ 메모리 할당
    ㄴ 드라이버 최적화
  ```

### 운영 관리
- **모니터링 및 백업**:
  ```
  1. 성능 모니터링
    ㄴ 리소스 사용량
    ㄴ 네트워크 상태
    ㄴ 이벤트 로그

  2. 백업 관리
    ㄴ 스냅샷 설정
    ㄴ 정기 백업
    ㄴ 복구 계획
  ```
